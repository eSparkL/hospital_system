## Git 提交规范

如果使用的是 VSCode 或者 JetBrains 旗下产品，在 Git 提交工具处编写提交说明时最好遵循  [约定式提交](https://www.conventionalcommits.org/zh-hans/v1.0.0/) 的内容。

常用前缀为：
- `feat`： 表示有新的功能和特性被加入；
- `docs`：表示更新了文档；
- `test`：表示修改或增加了测试用例；
- `refactor`：表示你重构了项目结构，包括重命名和移动文件位置；

例子：
```txt
feat: 增加了搜索功能
```
